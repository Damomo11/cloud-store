# Cloud Store Bot

基于 mineflayer 的 Minecraft 云仓库机器人。机器人一次只服务一名玩家，所有反馈默认通过 `/msg` 私聊发送，玩家库存以正版 UUID 记账。

## 环境

- Node.js 22 或更高版本
- Minecraft Java 服务器版本按 `config.json` 中的 `server.version` 配置
- 机器人账号需要能使用 `/home ck`、`/tpa 玩家名` 和 `/msg 玩家名 内容`

当前仓库的默认 Node 是 20 时不能直接运行最新版 mineflayer，请先安装 Node 22+。

## 安装

```bash
npm install
```

下载官方中文物品名：

```bash
npm run download:lang
```

## 配置

复制并编辑配置：

```bash
copy config.example.json config.json
```

关键配置：

- `server`: 服务器地址、端口、机器人账号、认证方式和 MC 版本
- `admins`: 管理员用户名，按你的服务器玩家名填写
- `commands.home`: 默认 `/home ck`
- `commands.tpa`: 默认 `/tpa {player}`，机器人主动请求传送到玩家身边
- `commands.tpaccept`: 默认 `/tpaccept {player}`，管理员请求传送到机器人位置时自动接受
- `storage.defaultCustomWarehouseQuotaSlots`: 自定义仓库默认额度
- `storage.maxCustomWarehousesPerPlayer`: 每个玩家最多可创建的自定义仓库数量，默认 3
- `web.enabled` / `web.host` / `web.port` / `web.dashboardUrl`: 内置 Web 面板开关、监听地址和展示地址
- `web.maxLoginFailures` / `web.loginFailWindowMs` / `web.loginBlockMs`: 登录密钥防爆破，默认 10 分钟内错 5 次封禁该 IP 10 分钟
- `web.trustProxy`: 是否信任穿透/反代传来的 `X-Forwarded-For` 真实 IP；公网反代需要按实际情况开启
- `timing.inventoryUpdateTimeoutMs`: 取物品点击后等待背包更新的时间，服务器卡顿时可调大
- `warehouse.defaultSyncRadius`: `!同步仓库` 的默认半径
- `debugChat`: 打印命令、回复、去重等命令相关日志
- `debugRawChat`: 打印原始公屏、私聊、系统聊天包；默认关闭，只有排查聊天格式时再打开
- `aliases`: 中文物品名到 `minecraft:item_id` 的映射
- `language.file`: 官方中文语言文件路径，默认 `data/zh_cn.json`

## 启动

```bash
npm start
```

## 内置 Web 面板

`npm start` 会同时启动机器人和 Web 面板。

默认地址：

```text
http://127.0.0.1:8787/
```

首次使用前，玩家需要先在游戏内私聊机器人设置登录密钥：

```text
设置密钥 1234
```

之后在网页首页输入这个密钥登录。每次重新设置都会覆盖旧密钥。

登录后只能看到自己的个人库存，以及自己加入的组织仓库库存。管理员登录后可以看到有库存的个人仓库、组织仓库和 `momo`。

Web 面板左侧提供“库存”和“交易记录”。库存页可以远程发起存物品、取物品任务，实际流程仍然由机器人在游戏内执行，玩家需要接受机器人的 `/tpa` 请求。

管理员登录后，左侧会额外显示“管理”按钮，可在同一个 8787 面板里查看总库存、库存异常、木桶内容、交易记录和别名管理，并进行数量设置、归属转移、物品别名维护等操作。

## 备用 Python 面板

旧的 Python 面板仍然保留，作为备用排查工具；日常不需要单独映射 8765 端口。

用户库存看板：

```bash
npm run dashboard
```

管理员本地查看器：

```bash
python data/warehouse_viewer.py
```

首次使用前，管理员需要站在仓库中心附近执行：

```text
!同步仓库 16
```

机器人会回 `/home ck`，扫描半径内箱子，记录箱子坐标和槽位。扫描出来多于账本的物品会划入 `momo`，账本多于实际库存的缺口会优先从 `momo` 扣除；如果 `momo` 不够，只记录异常，不扣普通玩家。

默认只扫描和操作木桶 `barrel`，普通箱子和潜影盒不会作为仓库容器。

## 玩家命令

```text
!存
!完成
!结束
!取 石英块64 石头128 红石块3
!查 石头
!额度
!设置密钥 1234
!创建仓库 仓库名1
!我的仓库
!仓库名1 添加成员 玩家ID
!仓库名1 加成员 玩家ID
!仓库名1 删除成员 玩家ID
!仓库名1 删成员 玩家ID
!仓库名1 添加管理员 玩家ID
!仓库名1 加管理员 玩家ID
!仓库名1 删除管理员 玩家ID
!仓库名1 删管理员 玩家ID
!仓库名1 仓库成员
!仓库名1 查 石头
!仓库名1 存
!仓库名1 取 石英块64 石头128
```

存物品流程：

1. 玩家输入 `!存`
2. 机器人回仓库并清空自身残留物，残留物记为 `momo`
3. 机器人 `/tpa 玩家名`
4. 玩家接受传送后，机器人检测到玩家在附近才开始收集
5. 玩家在 10 秒内丢物品，可用 `!完成` 或 `!结束` 提前结束
6. 机器人 `/home ck` 回仓库并入库记账

取物品流程：

1. 玩家输入 `!取 石英块64 石头128`
2. 机器人按玩家账本计算最大可取数量
3. 如果库存不足，会私聊提示，然后直接按最大数量出货
4. 机器人取出物品后 `/tpa 玩家名`
5. 玩家接受传送后，机器人检测到玩家在附近才开始丢物品
6. 如果物理箱子也不足，实际拿到多少就给多少，并且只扣实际给出的数量

查询只支持单一物品，避免刷屏。

自定义仓库和个人库存完全分开。每个玩家最多创建 3 个自定义仓库，仓库名全服唯一，不能包含空格，最多 15 个字。创建者自动成为仓库管理员，可以添加和删除成员。仓库名可以和玩家名相同，账本会用 `vault:仓库名` 区分，不会混到个人库存。

私聊机器人时可以省略命令前缀 `!`，例如 `存`、`查 石头`、`取 石头64`。公屏仍然必须使用 `!` 开头。

## 管理员命令

```text
!同步仓库
!同步仓库 16
!同步
!同步 16
!清空库存 确认
!查询 momo 石头
!查 momo 石头
!额度 玩家ID
!额度 仓库:仓库名
!加管理员 玩家ID
!设置额度 玩家ID 270
!设置额度 仓库:仓库名 270
!增加 玩家ID 石头 64
!减少 玩家ID 石头 64
```

`momo` 是未知归属和缓冲库存账户。机器人任务前发现自己背包有残留物时，会把残留物记入 `momo` 并正常入库。

`!清空库存 确认` 只用于测试清库：会清空库存账本、木桶位置、木桶槽位、存取记录和历史物品档案；不会清空配置别名、物品人工命名规则、机器人管理员、玩家、额度、自定义仓库和仓库成员。

## 设计约定

- 平时不启动扫描，也不在每次任务前扫描全仓库。
- 本地 SQLite 数据库位于 `data/cloud-store.sqlite`。
- 带 NBT 或组件的物品会生成独立指纹，尽量区分附魔、命名、耐久、潜影盒内容等特殊物品。
- 附魔书和附魔物品会尽量把组件里的附魔显示到名称中，例如 `经验修补一`、`耐久三`；查询和取出也可以用这些具体显示名匹配。
- 玩家改名后，库存仍跟随 UUID。若服务器事件中暂时拿不到 UUID，会临时用 `name:玩家名` 兜底。
- 机器人会向服务器声明 `zh_CN` 语言；命令解析使用 `data/zh_cn.json` 自动识别官方中文物品名，`aliases` 只需要补充简称或服务器自定义叫法。
- 玩家默认存储额度为 270 格，普通 64 堆叠物约等于 17280 个；存物品前如果未超额，本次存入允许正常完成，即使存完后超过额度。
- 自定义仓库使用独立 owner，默认额度由 `storage.defaultCustomWarehouseQuotaSlots` 控制，不会占用个人额度。
- `/tpa` 发出后，如果 15 秒内没有检测到目标玩家在附近，机器人会取消当前任务并提示玩家不要再接受这次传送请求。
- 机器人死亡后会自动重生并执行 `/home ck`。存物品过程中死亡会继续把背包物品记给原玩家；取物品过程中死亡会终止本次取货，把背包物品回收入库且不扣未交付数量。
